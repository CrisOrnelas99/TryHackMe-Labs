## 📘 TryHackMe Learning Path README
# 🚀 Overview

This repository tracks progress through multiple TryHackMe learning paths focused on web technologies, security, penetration testing, secure development, and offensive web operations. The goal is to build a strong foundation in how web applications work, how they are attacked, how they are defended, and how security is built into modern software and infrastructure from the start.

These labs cover both theory and hands-on practice, moving from core web concepts into practical security testing, defensive engineering, secure pipelines, and real-world attack methodology. The overall focus is on understanding systems deeply enough to identify weaknesses, reduce risk, improve secure design, and support resilient application development throughout the software lifecycle.

# 🌐 Web Fundamentals

This section focuses on the core concepts behind how the web works. Topics include websites, web servers, browsers, HTTP and HTTPS, requests and responses, methods, headers, cookies, sessions, DNS, subdomains, content discovery, authentication, and how web pages are built using HTML, CSS, and JavaScript. It also introduces how browser developer tools can be used to inspect source code, view requests, analyze scripts, and understand how client-side and server-side components interact.

The main goal is to build the technical base needed for later security work. Before testing or securing web applications, it is necessary to understand how pages are delivered, how users authenticate, how applications store state, and how client and server communicate.

# 🛡️ Security Engineering

This section focuses on defensive security concepts, secure architecture, and the engineering side of protecting systems. Learning includes risk reduction, secure configuration, system hardening, logging, monitoring, access control, vulnerability management, and security design principles. It also helps develop an understanding of how secure systems are designed, maintained, and improved over time rather than only how weaknesses are exploited.

The main value of this path is learning how security controls are implemented in real environments and how engineering decisions affect exposure, resilience, visibility, and long-term maintainability.

# 🔍 Web Application Penetration Testing

This section covers how to assess web applications for vulnerabilities through structured testing. Topics include reconnaissance, content discovery, subdomain enumeration, authentication weaknesses, session flaws, input validation issues, file upload abuse, access control problems, business logic flaws, and common web vulnerabilities. It also includes the use of browser tools, proxies, fuzzing tools, and manual testing techniques to identify insecure behavior.

The goal is to learn how to evaluate application security in a practical way by understanding application flow, mapping attack surface, analyzing requests and responses, and verifying whether controls are properly implemented.

# 💻 Jr Penetration Tester

This path develops practical penetration testing skills across common environments and attack surfaces. It introduces methodology, enumeration, exploitation, privilege escalation, service analysis, basic networking, web testing, and reporting. It is designed to strengthen technical problem solving while building confidence with tools, workflows, and lab-based assessment techniques.

The key outcome is learning how to move through an assessment in a structured way: identify targets, gather information, test weaknesses, verify impact, and document findings clearly.

# ⚙️ DevSecOps

This section focuses on integrating security into development and deployment workflows. Topics include CI/CD security, pipeline hardening, secrets management, dependency risk, IaC security, container security, cloud security basics, scanning automation, and secure software delivery practices. It connects development, operations, and security by showing how vulnerabilities can be prevented earlier in the lifecycle instead of only being found after deployment.

The main purpose is to understand how security becomes part of modern engineering processes, with emphasis on automation, validation, secure defaults, and reducing risk before code reaches production.

# 🎯 Web Application Red Teaming

This section focuses on offensive web application tradecraft with a more adversarial perspective. It builds on web fundamentals and penetration testing by emphasizing stealthier enumeration, chaining weaknesses, bypassing controls, abusing logic flaws, and approaching targets in a way that reflects real attack behavior. The focus is not just on finding isolated issues, but on understanding how separate weaknesses can be combined to achieve deeper access or greater impact.

This path strengthens attacker mindset, operational thinking, and the ability to analyze applications beyond checklist testing while also improving understanding of where stronger defensive design and validation are needed.

## 🧠 What This Repository Covers

This repository is intended to document learning across:

Web architecture and core internet concepts
HTTP/HTTPS communication and browser behavior
Authentication, sessions, cookies, and access control
Web reconnaissance and attack surface mapping
Common web vulnerabilities and testing methodology
Secure engineering and defensive implementation
Pentesting workflow and reporting discipline
DevSecOps concepts and secure delivery pipelines
Offensive web security and adversarial thinking
# 🛠️ Skills Being Built

Through these labs, the main skills being developed include understanding web application behavior, identifying exposed functionality, analyzing client-server communication, testing authentication and authorization, using common security tools, recognizing insecure patterns, and applying both offensive and defensive security concepts in practical scenarios.

There is also a strong focus on secure design awareness, validation of security controls, risk-based thinking, and understanding how security decisions affect application quality, deployment, and long-term maintainability.

# 📂 Purpose of This Repo

This repository serves as a learning log and reference point for TryHackMe lab progress across multiple structured paths. It is meant to organize concepts, notes, and walkthrough-based learning in a way that supports revision, portfolio growth, practical skill development, and a stronger understanding of secure application and infrastructure practices.

# ✅ Learning Goals

By completing these paths, the overall goal is to build a strong understanding of:

how web applications function
how common security weaknesses appear
how attackers discover and exploit those weaknesses
how systems are designed and improved to reduce exposure
how security fits into modern software development, testing, and operations

This creates a balanced foundation across both offensive and defensive security, with a strong emphasis on web applications, secure engineering practices, and practical cybersecurity skills.
